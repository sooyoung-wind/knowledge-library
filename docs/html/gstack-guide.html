gstack 설명서 및 시작 가이드

gstack은 Garry Tan이 공개한 AI 작업용 스킬 팩으로, Claude Code를 중심으로 CEO 리뷰, 엔지니어링 리뷰, 디자인 검토, 브라우저 QA, 문서 업데이트, 배포 검증까지 하나의 스프린트 흐름처럼 묶어 주는 도구 모음입니다.

이 페이지는 garrytan/gstack 저장소의 README와 공개 문서 링크를 바탕으로 2026년 4월 23일 기준 핵심 내용을 한국어 HTML 가이드로 재구성한 요약본입니다.

23+ 전문화된 워크플로 스킬
1 sprint 기획 → 구현 → 리뷰 → QA → 배포 흐름
Multi-agent OpenClaw, Codex 등 여러 에이전트 환경 지원

gstack이 하는 일

단순한 명령 모음이 아니라, AI가 팀처럼 움직이도록 역할과 순서를 강하게 부여하는 방식입니다.

누구에게 맞나

  • 혼자서도 제품을 빠르게 밀어붙이고 싶은 창업자나 테크 리드
  • Claude Code를 처음 쓰지만 빈 프롬프트보다 구조화된 역할이 필요한 사람
  • 코드 리뷰, QA, 문서 동기화, 배포 검증을 습관처럼 자동화하고 싶은 팀

핵심 철학

  • 가설과 문제 정의를 먼저 검증하고 코드는 그 다음에 쓴다
  • 스킬 하나하나보다, 스프린트 순서대로 연결되는 워크플로가 중요하다
  • 실제 브라우저 QA, 문서 갱신, 배포 후 검증까지 포함해 “출시”를 완결한다

빠른 시작

README가 제안하는 가장 짧은 체험 루트입니다.

설치

필수 조건은 Claude Code, Git, Bun 1.0+이며, Windows에서는 Node.js도 필요합니다.

git clone --single-branch --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack
cd ~/.claude/skills/gstack
./setup

문제 정의

/office-hours로 만들고 싶은 기능이 아니라 풀고 싶은 고통점부터 정리합니다.

계획 검토

/plan-ceo-review, /plan-eng-review, /plan-design-review로 범위와 설계를 다듬습니다.

구현 후 검증

/review로 변경 사항을 리뷰하고, /qa로 스테이징 URL을 실제 브라우저에서 테스트합니다.

권장 스프린트 흐름

README가 강조하는 gstack 사용 흐름을 실제 작업 순서 기준으로 묶으면 아래처럼 이해하면 쉽습니다.

1. 생각과 범위

/office-hours, /autoplan, /plan-ceo-review

문제 정의, 제품 포지셔닝, 범위 조정, 우선순위 결정을 담당합니다.

2. 설계와 품질

/plan-eng-review, /plan-design-review, /design-consultation

아키텍처, 엣지 케이스, UI 방향, 테스트 전략을 조기에 고정합니다.

3. 구현 보조

/design-shotgun, /design-html, /investigate

디자인 탐색, HTML 구체화, 버그 원인 조사 같은 실무 작업을 지원합니다.

4. 리뷰와 테스트

/review, /qa, /qa-only, /browse

변경 검토, 실제 브라우저 상호작용, 회귀 확인, 스크린샷 기반 검증을 맡습니다.

5. 배포와 운영

/ship, /land-and-deploy, /canary, /benchmark

PR 생성, 배포 후 상태 확인, 성능 회귀 감시를 이어서 실행합니다.

6. 문서와 학습

/document-release, /retro, /learn

출시 후 문서 갱신, 회고, 세션 간 학습 축적을 담당합니다.

설치와 팀 공유

개인 설치와 팀 모드의 목적이 조금 다릅니다.

개인 설치

로컬 사용자 계정에 gstack을 설치해 바로 사용할 수 있는 가장 간단한 방법입니다.

git clone --single-branch --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack
cd ~/.claude/skills/gstack
./setup

팀 모드

공유 저장소에 gstack 사용 정책을 추가해서 팀원이 같은 워크플로를 자동으로 쓰게 만드는 방식입니다.

(cd ~/.claude/skills/gstack && ./setup --team)
~/.claude/skills/gstack/bin/gstack-team-init required
git add .claude/ CLAUDE.md
git commit -m "require gstack for AI-assisted work"

OpenClaw 및 다른 에이전트

gstack은 Claude Code 전용이 아니라, 여러 에이전트 환경에 스킬을 설치하는 구조도 제공합니다.

OpenClaw와 함께

OpenClaw가 Claude Code 세션을 띄울 때, gstack 스킬을 불러 사용하도록 AGENTS.md에 지침을 추가하는 방식을 권장합니다.

  • 보안 감사: Run /cso
  • 코드 리뷰: Run /review
  • 기능 구현: /autoplan → implement → /ship
  • 기획: /office-hours → /autoplan

다른 호스트 지원

README에는 Codex, Cursor, OpenCode, Kiro, Hermes, Slate 등 여러 호스트별 설치 경로가 안내되어 있습니다.

./setup --host codex
./setup --host cursor
./setup --host opencode

Windows와 Codex 메모

README에서 특히 강조하는 운영체제 및 호환성 주의사항입니다.

Windows 11

  • Git Bash 또는 WSL 환경을 권장합니다.
  • Bun 외에 Node.js도 필요합니다.
  • 이유는 Windows에서 Playwright pipe transport 관련 Bun 이슈가 있어 browse 서버가 Node.js로 폴백하기 때문입니다.

Codex에서 스킬이 안 보일 때

README는 Codex 쪽 SKILL 설명이 오래된 경우 아래처럼 업데이트를 권장합니다.

cd ~/.codex/skills/gstack
git pull
./setup --host codex
